# 🔗 [프로젝트 소개](https://github.com/whynotsw-camp/wh02-3rd-1team-CHEONGYAGI/wiki)

<details>
  <summary>개발 목표 및 내용</summary>
  고령층을 포함한 디지털 취약계층을 대상으로 키오스크 음성 보조 챗봇 서비스를 구현하여, 복잡한 청약 제도의 접근성을 향상시키고 효율적인 청약 프로세스를 지원하고자한다. <br>
  컴퓨터 비전 기술로 사용자의 연령대를 추정하고, 이를 바탕으로 접근성 향상 기능을 먼저 제안하는 선제적 지원 방식을 도입하여 차별점을 두었습니다.
</details>
<details>
  <summary>주요 타겟층</summary>

  - 고령층(65세 이상) 및 디지털 취약계층 <br>
  - 장애인 및 신체적 제약이 있는 사용자<br>
  - 패스트푸드점·카페 등 키오스크 활용 업장 & 관리자<br>

</details>
<details> <summary>핵심 기능</summary>

  - 웹캠을 통한 컴퓨터 비전으로 사용자의 연령대 추정<br>
  - 65세 이상 사용자를 위한 '어르신 맞춤 모드' 자동 전환<br>
  - 음성 안내(TTS), 직관적 음성 명령(STT) 제공<br>
  - 음성 기반 주문 처리 시스템<br>
  - 메뉴 검색, 필터링, 장바구니 담기, 결제까지 음성만으로 진행<br>
  - 알레르기·비건 필터링, 영양 정보 확인 기능 제공<br>
  - 자연어 대화 기반 추천 및 주문 확인 지원<br>
  - 동일한 menu.json 데이터 기반으로 확장성 확보<br>

</details>
<details> <summary>보조 기능</summary>
  - 답변 듣기(TTS) 버튼 : 모든 텍스트를 음성으로 안내<br>
  - 메뉴 필터링 기능 : 알레르기, 채식, 저염 등 개인 맞춤형 필터 제공<br>
  - 주문 내역 확인 기능 : 음성 또는 시각적으로 주문 내역 검증<br>
  - menu.json 기반 확장성 : 데이터 교체만으로 프랜차이즈별 손쉬운 적용<br>
  - 다국어 지원(확장 예정) : 외국인 고객을 위한 다국어 STT/TTS 지원 가능성<br>
  - 웹 기반 MVP : 태블릿(iPad 등)에서 바로 시연 가능<br>
</details>

## 📁 프로젝트 구조

```
integrated_kiosk_project/
├── 01_face_recognition_app.py    # 얼굴 인식 앱 (포트 8501)
├── 02_normal_kiosk_app.py        # 일반 키오스크 앱 (포트 8502)
├── 03_voice_kiosk_app.py         # 음성 주문 키오스크 앱 (포트 8503)
├── menu_data.py                  # 메뉴 데이터
├── requirements.txt              # 필요한 패키지 목록
├── public/                       # 메뉴 이미지 파일들
└── README.md                     # 이 파일
```

## 🚀 실행 방법

### 1. 환경 설정

먼저 필요한 패키지들을 설치합니다:

```bash
pip install -r requirements.txt
```

### 2. 환경 변수 설정

`.env` 파일을 생성하고 다음 내용을 추가합니다:

```env
# Face++ API (얼굴 인식용)
FACE_API_KEY=your_face_api_key_here
FACE_API_SECRET=your_face_api_secret_here

# OpenAI API (음성 주문용)
OPENAI_API_KEY=your_openai_api_key_here
```

### 3. 앱 실행 (권장 방법)

**Windows에서 한 번에 모든 앱 실행:**
```bash
run_all.bat
```

**macOS/Linux에서 한 번에 모든 앱 실행:**
```bash
chmod +x run_all.sh
source venv/bin/activate && ./run_all.sh
```

이 명령어를 실행하면:
- **얼굴 인식 앱 (포트 8501)**: 포어그라운드에서 실행되며 브라우저가 자동으로 열림
- **일반 키오스크 (포트 8502)**: 백그라운드에서 실행 (브라우저 자동 열림 없음)
- **음성 주문 키오스크 (포트 8503)**: 백그라운드에서 실행 (브라우저 자동 열림 없음)

### 4. 수동 실행 (개별 앱 테스트용)

개별 앱을 테스트하고 싶다면 각각 터미널에서 실행할 수 있습니다:

**얼굴 인식 앱:**
```bash
streamlit run 01_face_recognition_app.py --server.port 8501
```

**일반 키오스크:**
```bash
streamlit run 02_normal_kiosk_app.py --server.port 8502
```

**음성 주문 키오스크:**
```bash
streamlit run 03_voice_kiosk_app.py --server.port 8503
```

## 🔄 워크플로우

1. **시작**: `run_all.bat` 실행으로 모든 앱이 자동으로 시작됨
2. **얼굴 인식 (포트 8501)**: 사용자의 얼굴을 인식하여 나이를 추정
3. **자동 라우팅**: 
   - 65세 미만 → 일반 키오스크 (포트 8502)로 자동 이동
   - 65세 이상 → 음성 주문 키오스크 (포트 8503)로 자동 이동
4. **주문 및 결제**: 각 키오스크에서 메뉴 선택 및 결제 진행
5. **자동 복귀**: 결제 완료 후 5초 뒤 자동으로 얼굴 인식 페이지로 돌아감

## 🛠️ 주요 기능

### 얼굴 인식 앱 (01_face_recognition_app.py)
- 실시간 카메라를 통한 얼굴 인식
- Face++ API를 사용한 나이 추정
- 자동 라우팅 기능

### 일반 키오스크 (02_normal_kiosk_app.py)
- 터치 기반 메뉴 선택
- 알레르기 및 식단 필터링
- 장바구니 관리
- 결제 처리

### 음성 주문 키오스크 (03_voice_kiosk_app.py)
- 음성 인식 (STT) 및 음성 합성 (TTS)
- AI 챗봇을 통한 자연어 주문
- LangChain 기반 대화 처리
- 시각적 메뉴와 음성 주문 병행

## 📋 메뉴 정보

- **메인**: 버거, 랩, 보울, 샐러디
- **사이드**: 감자튀김, 치즈스틱
- **음료**: 콜라, 사이다
- **디저트**: 아이스크림, 팥빙수

각 메뉴에는 알레르기 정보, 식단 정보, 가격, 조리시간 등이 포함되어 있습니다.

## 🔧 문제 해결

### run_all.bat 실행 문제
- Windows 환경에서만 실행 가능
- PowerShell이나 명령 프롬프트에서 실행
- 권한 문제 시 관리자 권한으로 실행

### 음성 인식 문제
- 마이크 권한 확인
- PyAudio 설치 문제 시: `pip install pipwin` 후 `pipwin install pyaudio`

### 얼굴 인식 문제
- 카메라 권한 확인
- Face++ API 키 설정 확인

### 포트 충돌
- 다른 포트 사용: `--server.port 8504` 등으로 변경
- 백그라운드 앱 접속: 브라우저에서 `http://localhost:8502` 또는 `http://localhost:8503` 직접 접속

<p> 이번 프로젝트는 '디지털 소외 계층을 돕는다'는 의미 있는 목표 덕분에 개발 과정 내내 보람을 느낄 수 있었습니다. 기술적으로는 GitHub를 이용한 협업, 터미널, API 키 관리, Streamlit을 활용하여 배웠던 개발 기술들을 직접 경험하는 값진 시간이었습니다. 기술적 성장, 다양한 경험과 개발의 동기를 함께 얻은 좋은 경험이었습니다.
</p>
<br>

<h3>김동은</h3>
<p>프로젝트 구조를 세부적으로 짜지 못해 효율적인 업무 분담을 하지 못한 것 같아 아쉬웠고, 세부적인 구조를 짜기 위해선 폭넓고 깊은 지식이 필요한 것 같아 개인적인 부족함을 많이 느낌.</p>
<br>


<h3>김두현</h3>
<p>이론으로 배우던 내용을 실제로 구현하기 위한 시도에 의미를 둠. 체계적인 설계 필요성을 체감함.
</p>
<br>


<h3>김민지</h3>
<p>아무리 AI가 코드를 대신 작성해 준다 해도 결국 개발자가 그 코드와 구조를 충분히 이해하지 못하면 한 단계씩 나아갈 때마다 벽에 부딪히게 된다는 것을 깨달았다. 막판에 시간이 부족해 제대로 구현하지 못한 알레르기 필터 기능과 외국인 버전까지 구현한다면 완성도를 더 높일 수 있을 것 같다.</p>
<br>


<h3>전지성</h3>
<p>프로젝트를 진행하면서 기초 개념을 튼튼히 해야 응용이 수월하다는 걸 깨달았고, 이번 경험을 통해 소통의 중요성도 실감했다. 앞으로는 결과에 급급하기보다 기본기를 먼저 다지고, 매일 핵심 개념을 복습하는 습관을 실천하겠다. 더불어 프로젝트에 적극적으로 참여하며 소통에도 힘써야겠다.</p>
<br>


<h3>오현영</h3>
<p>프로젝트를 진행하면서 LLM과 LangChain을 직접 활용해볼 수 있어 의미 있는 경험이었습니다. 다만, 짧은 기간으로 인해 프론트엔드와 백엔드를 완전히 연동하지 못한 점은 아쉬움으로 남습니다.</p>
